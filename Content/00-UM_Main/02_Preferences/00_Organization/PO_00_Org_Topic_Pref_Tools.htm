Individual Preference Tools

Renaming a Preference Instance

Mousing over the current preference name, displays the edit button. Click it to make the name field editable.

Enable and Disable

Clicking the disable button disables the preference to which it is attached. The preference will be grayed out. The enable button appears only on a disabled preference instance. Click it to enable the preference instance.

Delete

The deletes. There is no un-delete. If you are not sure about keeping or discarding a preference instance, disable it until you are certain.

The Time Intervals filter allows you to limit the operation of your preference to one or more time ranges:

Preference Edit Filters

Notice the use of the Next Day check boxes to cater for 24 hour spill-over. They effectively enable you to set time ranges beyond midnight of the current day. This example creates two time ranges:

  1. From 13:00 each day to 01:00 the following day
  2. From 06:00 to 10:00 each day

By default, the preference instance will be operative only within the range(s) that you specify.

You can reverse the ranges using the Outside Ranges button. In our example, this would cause the preference to be operative as follows:

  1. From the beginning of operations for the schedule to 13:00 each day and then from 01:00 to the end operations time for the schedule.
  2. From the beginning of operations for the schedule to 06:00 and from 10:00 to the end operations time for the schedule.

Both range methods may have overlapped time ranges, which will be handled correctly by the system.
