Deadheads Limitation

Work-flow to Use Deadheads Limitation

  • Use Load Template to set up the preference for several conditions
  • Repeat Load Template for additional instances of any template

Preference Overview

This is a set of complimentary preference instances that set distance limits for deadheads and depot pull out and depot pull in trips. It is intended to minimize deadhead distances to keep the vehicles close to their “home” depots.

Reduce depot checks

Purpose:

Reduces the duration of the depot optimization by minimizing the number of return depot “candidates”.

Prerequisites: None.

Opening Dialog:

Points to note:

  • Depot distance limit: This is the maximum distance for a depot pull out/pull in
  • If you set it to zero, vehicles will always return to their "home" depot
  • The unit of distance (Km, miles) depends on your location

Minimize Vehicle Cruise Preference

Purpose:

Set the maximum distance for an exit depot deadhead taking the vehicle further away from its "home" depot with a penalty.

Prerequisites: None.

Opening Dialog:

Points to note:

  • This preference is a flexible version of Minimize Vehicle Cruise Restriction
  • Deadhead distance limit: This field defaults to 30 and is flexible with a Penalty as set in the following field
  • The unit of distance (Km, miles) depends on your location
